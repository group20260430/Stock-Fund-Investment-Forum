## 一、设计总则

### 1.1 设计目标

面向股票、基金投资者的社区论坛，UI 设计遵循以下原则：

| 原则 | 说明 |
|------|------|
| **信息密度适中** | 投资者需要快速浏览大量信息，布局应紧凑但不拥挤 |
| **数据可视化优先** | 股票行情、热榜排名等数据以卡片、列表、涨跌色清晰呈现 |
| **操作路径短** | 核心操作（发帖、评论、点赞）不超过 2 次点击 |
| **响应式适配** | 桌面端为主要使用场景，同时适配平板和手机 |
| **专业感** | 配色稳重、排版清晰，符合金融投资类产品的调性 |

### 1.2 参考设计风格

采用现代 SaaS / 社区论坛风格，参考：
- **雪球** — 投资社区的内容布局、行情卡片
- **Reddit** — 帖子列表的层级评论结构
- **Tailwind CSS 官方模板** — 简洁的组件设计

## 二、视觉规范

### 2.1 色彩系统

以**深青色（Teal）** 为主色调，传递"稳健、专业、成长"的品牌感。

```
┌─────────────────────────────────────────────────────────────┐
│  主色 (Primary)                                              │
│  teal-600   #0F766E   按钮、链接、活跃态                      │
│  teal-700   #0F766E   按钮悬停态                              │
│  teal-50    #F0FDFA   选中背景                                │
├─────────────────────────────────────────────────────────────┤
│  中性色 (Neutral)                                            │
│  gray-900   #111827   侧边栏背景、深色区域                     │
│  gray-800   #1F2937   侧边栏悬停                              │
│  gray-700   #374151   次要标题                                │
│  gray-600   #475569   正文文字                                │
│  gray-500   #64748B   辅助文字、图标                           │
│  gray-300   #CBD5E1   侧边栏文字                              │
│  gray-200   #E5E7EB   边框、分割线                             │
│  gray-100   #F4F6F8   页面背景                                │
│  white      #FFFFFF   卡片背景                                │
├─────────────────────────────────────────────────────────────┤
│  功能色 (Functional)                                          │
│  red-600    #DC2626   下跌、删除、违规（涨跌色体系）            │
│  red-700    #B91C1C   错误状态文字                             │
│  green-500  #22C55E   上涨、通过、成功                          │
│  amber-500  #F59E0B   警告、待审核                              │
│  blue-500   #3B82F6   链接、信息提示                           │
└─────────────────────────────────────────────────────────────┘
```

**涨跌色规则（金融行业惯例）：**
- A股市场：红涨绿跌（`red-600` 涨，`green-500` 跌）
- 其他场景：绿色表示正向操作（成功、通过），红色表示负向操作（失败、拒绝）

### 2.2 字体排版

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| H1 | 28px | Bold (700) | 页面主标题 |
| H2 | 22px | Semibold (600) | 区块标题、行情指数数值 |
| H3 | 18px | Semibold (600) | 帖子标题、卡片标题 |
| Body | 16px | Regular (400) | 正文内容 |
| Body-Small | 14px | Regular (400) | 辅助信息 |
| Caption | 13px | Regular (400) | 元数据（时间、统计数） |
| Small | 12px | Regular (400) | 标签、徽章 |

**字体族：** `Inter, "PingFang SC", "Microsoft YaHei", Arial, sans-serif`

- Inter：英文和数字（等宽数字，便于阅读金融数据）
- PingFang SC / Microsoft YaHei：中文回退字体

### 2.3 间距系统

基于 Tailwind CSS 默认间距（4px 步长）：

| Token | 值 | 用途 |
|-------|-----|------|
| `gap-2` | 8px | 紧密元素间距（标签、图标） |
| `gap-3` | 12px | 列表项间距 |
| `gap-4` | 16px | 卡片内间距、表单间距 |
| `gap-5` | 20px | 区块间距、工具栏间距 |
| `gap-6` | 24px | 页面级区块间距 |
| `gap-8` | 32px | 页面内容区 padding |

### 2.4 圆角与阴影

| 元素 | 圆角 | 阴影 |
|------|------|------|
| 卡片 | `rounded-lg` (8px) | `shadow-sm` |
| 按钮 | `rounded-md` (6px) | 无 |
| 输入框 | `rounded-md` (6px) | 无 |
| 头像 | `rounded-full` (圆形) | 无 |
| 模态框 | `rounded-xl` (12px) | `shadow-lg` |
| 导航链接 | `rounded-md` (6px) | 无 |

## 三、全局布局

### 3.1 整体结构

采用**侧边栏 + 主内容区**的两栏布局，桌面端侧边栏固定 240px，移动端侧边栏隐藏。

```
┌──────────┬─────────────────────────────────────────────┐
│          │  ┌──────────────────────────────────────┐   │
│          │  │  顶部工具栏 (标题 + 操作按钮)          │   │
│          │  └──────────────────────────────────────┘   │
│          │                                             │
│  侧边栏  │  ┌──────────────────────────────────────┐   │
│ 240px    │  │                                       │   │
│          │  │        主内容区域                       │   │
│  · Logo  │  │    (帖子列表 / 页面内容)                │   │
│  · 导航  │  │                                       │   │
│  · 板块  │  │                                       │   │
│          │  └──────────────────────────────────────┘   │
│          │                                             │
└──────────┴─────────────────────────────────────────────┘
```

**CSS 实现：**
```css
.app-shell {
  display: grid;
  grid-template-columns: 240px 1fr;
  min-height: 100vh;
}
```

### 6.2 路由守卫

```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')

  if (to.meta.requiresAuth && !token) {
    // 未登录 → 跳转登录页，登录后回跳
    next({ name: 'login', query: { redirect: to.fullPath } })
  } else if (to.meta.requiresAdmin) {
    // 检查用户角色
    const user = getUserFromToken(token)
    if (user?.role !== 'admin') {
      next({ name: 'home' }) // 非管理员重定向到首页
    } else {
      next()
    }
  } else {
    next()
  }
})
```

| `api/search.js` | /search, /hot, /feed | 搜索、热榜、Feed |
| `api/admin.js` | /admin/* | 审核、用户管理、统计 |

### 7.2 请求/响应处理规范

**请求封装 (`api/request.js`)：**

```javascript
// 统一响应解包：后端返回 { code, message, data }，前端直接拿 data
// Token 过期 → 自动清除 token → 跳转登录页
// 网络错误 → 统一 Toast 提示
// 支持 GET/POST/PUT/PATCH/DELETE
```

**后端统一响应格式：**
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

前端 `request.js` 自动提取 `data` 字段，业务代码只需处理业务数据。

## 九、交互规范

### 9.1 即时反馈

| 操作 | 反馈方式 |
|------|---------|
| 点赞/取消点赞 | 图标颜色即时切换 + 数字动画（乐观更新） |
| 收藏/取消收藏 | 图标切换 + Toast 提示 |
| 关注/取消关注 | 按钮文字/样式即时切换 |
| 表单提交 | 按钮变为 loading 态，禁止重复点击 |
| 删除操作 | 先弹出确认对话框，再执行删除 |
| 发帖成功 | 跳转到帖子详情页 |
| 评论成功 | 评论即时追加到列表底部 |

### 9.2 表单校验

| 字段 | 校验规则 | 触发时机 |
|------|---------|---------|
| 手机号 | 11位数字，1开头 | 失焦时 |
| 密码 | 8-32位，含字母+数字 | 失焦时 |
| 验证码 | 6位数字 | 提交时 |
| 昵称 | 2-20字符 | 失焦时 |
| 帖子标题 | 1-120字 | 失焦时 |
| 帖子内容 | 非空，最少10字 | 提交时 |
| 评论内容 | 非空，最多2000字 | 提交时 |

- 验证失败：输入框红色边框 + 下方红色提示文字
- 验证通过：输入框绿色边框（可选）

### 9.3 Toast 通知

位置：页面顶部居中，自动消失（3秒）

| 类型 | 颜色 | 图标 | 场景 |
|------|------|------|------|
| success | 绿色 | ✅ | 操作成功（发布、保存） |
| error | 红色 | ❌ | 操作失败（提交失败） |
| warning | 黄色 | ⚠️ | 警告（即将超时） |
| info | 蓝色 | ℹ️ | 信息提示（新消息） |

### 9.4 空状态与错误状态

每个数据列表/页面都必须覆盖：

```
加载中 → 骨架屏（3-5个占位卡片，脉冲动画）
   ↓
有数据 → 正常渲染
无数据 → 空状态插画 + "暂无内容" + 操作引导
   ↓
加载失败 → 错误图标 + 错误信息 + [重试] 按钮
```

## 十、前端性能优化

### 10.1 优化策略

| 策略 | 应用场景 |
|------|---------|
| **路由懒加载** | 所有页面组件使用 `() => import(...)` 动态导入 |
| **组件按需加载** | 富文本编辑器、图表等重量组件按需加载 |
| **列表虚拟滚动** | 超长评论列表、用户列表使用虚拟滚动 |
| **图片懒加载** | 帖子中图片使用 `loading="lazy"` + 占位图 |
| **请求去重** | 同一接口短时间内不重复请求 |
| **乐观更新** | 点赞/收藏等操作先更新UI再发请求，失败回滚 |
| **Token 缓存** | JWT Token 存储在 localStorage，减少登录请求 |

### 10.2 打包优化

- Vite 自动 Tree Shaking
- 第三方库按需引入（如只引入用到的图表类型）
- 静态资源（图片/字体）使用 CDN 或放到 `public/`
